# Docker Setup for Recaply API

This directory contains the Docker setup for Recaply, including the API, reverse proxy, data stores, and monitoring stack.

## Services Overview

### Core Application
- API (FastAPI): main backend running with Uvicorn
- Nginx: reverse proxy in front of API

### Databases and Storage
- MongoDB: primary document database
- Qdrant: vector database
- Redis: cache/session store

### Monitoring
- Prometheus: metrics scraping
- Grafana: dashboards
- MongoDB Exporter: MongoDB metrics exporter
- Redis Exporter: Redis metrics exporter

## Environment Setup

Before starting services, create required env files:

```bash
cd docker/env
cp .env.example.app .env.app
cp .env.example.grafana .env.grafana
cp .env.example.mongodb .env.mongodb
cp .env.example.redis .env.redis
```

Optional project-level env file:

```bash
cd docker
cp .env.example .env
```

## Setup Instructions

### 1. Set up environment files

Create your env files from the examples:

```bash
# Create all required .env files from examples
cd docker/env
cp .env.example.app .env.app
cp .env.example.grafana .env.grafana
cp .env.example.mongodb .env.mongodb
cp .env.example.redis .env.redis
```

### 2. Start the services

Run from project root:

```bash
docker compose -f docker/docker-compose.yml up --build -d
```

Or run from inside `docker/`:

```bash
cd docker
docker compose up --build -d
```

To start only specific services:

```bash
docker compose -f docker/docker-compose.yml up -d api nginx mongodb qdrant redis
```

If you encounter connection issues, start databases first and then app services:

```bash
# Start databases first
docker compose -f docker/docker-compose.yml up -d mongodb qdrant redis
# Wait for databases to be ready
sleep 30
# Start API and infra services
docker compose -f docker/docker-compose.yml up -d --build api nginx prometheus grafana mongodb-exporter redis-exporter
```

In case deleting all containers and volumes is necessary:

```bash
docker compose -f docker/docker-compose.yml down -v --remove-orphans
```

### 3. Access the services

- API: http://localhost:8000
- API docs: http://localhost:8000/docs
- Nginx proxy: http://localhost
- Prometheus: http://localhost:9090
- Grafana: http://localhost:3000
- Qdrant dashboard: http://localhost:6333/dashboard

## Volume Management

Docker volumes are used to persist data generated by containers.

1. List all volumes:

```bash
docker volume ls
```

2. Inspect a volume:

```bash
docker volume inspect <volume_name>
```

List files in a volume:

```bash
docker run --rm -v <volume_name>:/data busybox ls -l /data
```

3. Remove a volume:

```bash
docker volume rm <volume_name>
```

4. Prune unused volumes:

```bash
docker volume prune
```

5. Backup volume for migration:

```bash
docker run --rm -v <volume_name>:/volume -v $(pwd):/backup alpine tar cvf /backup/backup.tar /volume
```

6. Restore volume from backup:

```bash
docker run --rm -v <volume_name>:/volume -v $(pwd):/backup alpine sh -c "cd /volume && tar xvf /backup/backup.tar --strip 1"
```

7. Remove all volumes:

```bash
docker volume rm $(docker volume ls -q)
```

## Monitoring

### API Metrics

API exposes Prometheus metrics on `/metrics`, including:
- request count
- request latency
- status codes

Prometheus is configured to scrape these metrics.

### Visualizing Metrics in Grafana

1. Open Grafana: http://localhost:3000
2. Login with credentials from `docker/env/.env.grafana`
3. Add Prometheus data source (`http://prometheus:9090` from inside Docker network)
4. Import dashboards

## Development Workflow

The API container runs from code copied at build time. After source changes, rebuild API image:

```bash
docker compose -f docker/docker-compose.yml up -d --build api
```

## Troubleshooting

### Connection Errors

If you see connection errors during startup:

1. Start databases first, then API.
2. Check status:

```bash
docker compose -f docker/docker-compose.yml ps
```

3. Check logs:

```bash
docker compose -f docker/docker-compose.yml logs --tail=100 api
docker compose -f docker/docker-compose.yml logs --tail=100 mongodb
docker compose -f docker/docker-compose.yml logs --tail=100 qdrant
docker compose -f docker/docker-compose.yml logs --tail=100 redis
```

4. Restart API after dependencies are up:

```bash
docker compose -f docker/docker-compose.yml restart api
```
